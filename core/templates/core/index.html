VulnLab Django (Intentionally Insecure)

Use only in local isolated environment.

Quick Start

  1. Call /seed/ to load demo users and records.
  2. Test each endpoint below with browser, curl, or your scanner.

Endpoints

  • /seed/ - unauthenticated data bootstrap
  • /login-unsafe/ (POST) - SQLi + weak auth
  • /register-mass/ (POST) - mass assignment (set role/token directly)
  • /impersonate/?user_id=1 - auth bypass / session impersonation
  • /admin-panel/?admin=1 - admin auth bypass by query param
  • /profile/?user_id=1 - IDOR + sensitive data exposure
  • /records/1/ - IDOR on records
  • /users-search/?q=... - SQL injection (search)
  • /users-leak/ - full user leak + permissive CORS
  • /xss-reflected/ - reflected XSS
  • /guestbook/ - stored XSS + no CSRF
  • /cmd/?cmd=id - command injection / RCE
  • /eval/?expr=... - arbitrary eval RCE
  • /read-file/?path=/etc/hosts - path traversal / local file read
  • /pickle/?payload=... - insecure deserialization
  • /reset-password/?user_id=1 - predictable reset token
  • /go/?next=https://example.com - open redirect
  • /fetch/?url=... - SSRF
  • /upload/ (POST multipart) - unrestricted upload
  • /extract-tar/ (POST multipart) - unsafe TAR extraction (Zip Slip style)
  • /change-password/ - state change by GET, no auth
  • /ssti/?tpl=... - template injection
  • /debug-env/ - environment leak
